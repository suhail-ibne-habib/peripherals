Bus Interface Unit (BIU)

The BIU handles all transfers of data and addresses on the buses for the execution unit. It sends out addresses, fetches instructions from memory, reads data from ports and memory, and writes data to ports and memory.

Pipelining

Pipelining is a technique where multiple instructions are overlapped in execution. The BIU uses pipelining to pre-fetch instructions and store them in the instruction queue for faster execution.

Execution Unit (EU)

The Execution Unit (EU) is responsible for decoding and executing instructions. It processes data provided by the BIU and stores results in internal registers like AX, BX, CX, and DX.

I/O Interfacing

I/O interfacing involves transferring data between input-output devices and the microprocessor. The three methods are program-controlled I/O, interrupt-driven I/O, and hardware-controlled I/O (DMA).

Memory Banking

Memory banking is a technique used in the 8086 microprocessor to access two consecutive memory locations in one cycle. It divides memory into even and odd banks to transfer 16 bits of data simultaneously.

Memory-Mapped I/O

Memory-mapped I/O treats I/O devices as if they were memory locations. The same address space is used for both memory and I/O devices.

Isolated I/O

Isolated I/O uses separate address spaces for memory and I/O devices. Special instructions (IN and OUT) are used to access I/O devices.

Bus

A bus is a communication system that transfers data between components inside a computer or between computers. The 8086 microprocessor uses three types of buses: address bus, data bus, and control bus.

Port

A port is a connection point or interface between a computer and an external or internal device. It can be used to connect devices like printers, modems, and hard drives.

Interfacing

Interfacing is the process of connecting the 8086 microprocessor with peripherals like memory and I/O devices to perform operations. It includes memory interfacing and I/O interfacing.
