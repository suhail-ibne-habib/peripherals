Memory Banking in 8086

The 8086 processor provides a 16-bit data bus. It is capable of transferring 16 bits in one cycle, but each memory location is only of a byte (8 bits). Therefore, we need two cycles to access 16 bits (8 bits each) from two different memory locations.

The solution to this problem is Memory Banking.

Even Bank (Lower Bank)

Contains even addresses and stores the lower byte of data.

Odd Bank (Higher Bank)

Contains odd addresses and stores the higher byte of data.

Memory Banking

Memory banking is a technique used in the 8086 microprocessor to access two consecutive memory locations in one cycle. It divides memory into even and odd banks to transfer 16 bits of data simultaneously.
